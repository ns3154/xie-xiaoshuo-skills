# 小说封面

## 1. 定位

为中文网文项目生成或迭代封面。封面任务不改正文，不改大纲，除非用户明确要求同步包装文案。

- 缺书名或笔名时必须先问。
- 优先读取项目简介、题材、卖点、主角身份、风格禁忌和目标平台。
- 封面要服务读者第一眼：题材、主角、氛围、核心卖点必须清楚。
- 不生成与作品类型相反的封面调性。

## 2. 输入

尽量收集：

- 书名、作者名。
- 平台和尺寸要求。
- 题材标签、目标读者、核心卖点。
- 主角外观、关键物件、场景、时代质感。
- 禁止元素、参考图、希望避开的竞品风格。
- 输出目录，默认 `covers/<书名>/` 或项目内 `封面/`。

## 3. 流程

1. 读取项目信息，提炼封面信号。
2. 写封面设计 brief：画面主体、背景、色彩、字体、情绪、平台识别点。
3. 检查本机出图能力：`GPT_IMAGE_API_KEY`、`curl`、`jq`、`base64`。
4. 生成含书名和作者名的封面图，必要时输出多版。
5. 做质量检查：标题可读、署名可读、题材明确、无明显错字、无主体遮挡、移动端缩略图能识别。
6. 按平台尺寸导出或说明现有尺寸。

详细出图流程读取 `references/cover-generation.md`。

## 4. 质量边界

- 书名必须是第一视觉信号，不能只放在角落。
- 笔名要清楚，但不能压过书名。
- 主体、时代、题材必须明确；不要做纯抽象氛围图。
- 封面不使用正文剧透级反转，除非用户明确要求。
- 若生成模型文字出错，优先迭代文字区域，不把错字图当成成品。
