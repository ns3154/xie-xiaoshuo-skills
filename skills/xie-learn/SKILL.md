# 扫榜与拆文学习

## 1. 定位

只吸收市场和对标作品的方法层，不把外部作品的桥段、人物关系、设定名词和原文表达搬进本书。

- 当前榜单、平台趋势、书籍数据属于易变信息；需要最新判断时必须联网或调用可用脚本核验。
- 对标资料不能覆盖本书正文事实、文风禁忌、人物声口和用户硬边界。
- 输出应落到“本书可执行的方法”，而不是泛泛夸书或复述剧情。

## 2. 扫榜

扫榜时优先记录数据质量：

```markdown
## 数据质量
- 平台：
- 采集时间：
- 榜单入口：
- 样本数量：
- 字段完整度：
- 是否登录态：
- 可能偏差：
```

观察维度：

- 题材、卖点、主角身份、开局承诺、金手指、爽点类型。
- 章节标题、简介、封面、标签、更新节奏、字数段。
- 前三章钩子、压迫时长、爆点间隔、情绪模块。
- 读者预期：想看什么、怕什么、什么时候弃。

如本机已有 `story-long-scan` 或 `story-short-scan` 的采集脚本，可借用脚本；输出仍按本技能沉淀到本项目。

## 3. 拆文

拆文产物默认放：

```text
拆文库/{书名}/
```

推荐结构：

```text
拆文库/{书名}/00-拆文总览.md
拆文库/{书名}/01-黄金三章.md
拆文库/{书名}/剧情/节奏.md
拆文库/{书名}/剧情/情绪模块.md
拆文库/{书名}/人物/人物关系.md
拆文库/{书名}/设定/爽点机制.md
拆文库/{书名}/文风.md
拆文库/{书名}/可迁移方法.md
```

拆文先看黄金三章，再决定是否继续全量拆。不要在没读正文的情况下只凭简介写深度结论。

## 4. 可迁移层

拆文后只沉淀这些：

- 情绪交付：压、爆、观众反应、余波。
- 节奏：章节定位、高压/低压错落、断章位置、爆点密度。
- 结构：卷目标、阶段目标、反派递进、收益与代价。
- 人物：声口差异、关系拉扯、信息差、误解与隐瞒。
- 文风：句长、段落、动作密度、对白比例、历史质感或类型质感。
- 商业包装：书名、简介、封面、标签、前 3 章承诺。

禁止迁移：

- 原文句子、桥段组合、关键反转、人物关系、专名、设定命名、章节标题模式的近似照搬。

## 5. 写入本项目

对标结论默认写入：

```text
资料/写作记忆/13-对标与商业参考.md
```

每条建议必须写清：

- 来源作品或榜单。
- 观察到的技法。
- 本书可用的改法。
- 本书不可用或必须避开的地方。
- 会影响哪些章节、人物、节奏或包装材料。

远端 `oh-story-claudecode` 的新方法可吸收：短篇格式/技法/去 AI 味拆分、长篇章节定位、对标节奏迁移、情绪模块库、字数预算和确定性 fallback。吸收时必须改写成本地项目规则。
