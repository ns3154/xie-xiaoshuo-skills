# 导入已有小说

## 1. 定位

把已有小说反向整理成可继续写、可审稿、可修订的项目结构。导入不是改写，默认不改变原文。

- 先确认源文件、目标目录、是否允许覆盖、是否只分析不落盘。
- 原稿保留备份；不要边导入边擅自润色。
- 章节拆分和排序按正文实际章号数值处理。
- 长篇、短篇、合集、散稿按材料形态选择不同粒度。

## 2. 输入确认

需要确认或自动识别：

- 源文件：`.txt`、`.md`、`.docx`、目录、压缩包或多个章节文件。
- 目标：完整写作项目、只生成拆文报告、只补写作记忆。
- 正文目录名：默认 `正文/`。
- 章节标题格式：默认 `第x章 章节名`。
- 作品类型：长篇、短篇、系列短篇、完本、半成品。
- 是否保留原文件名，是否需要重排章号。

如果用户给出明确路径和目标，直接执行；只有覆盖风险或标题解析不确定时才停问。

## 3. 导入流程

1. 建立导入索引：源文件、章节标题、章号、字数、异常项。
2. 生成或整理 `正文/`，每章一个 `.txt`，第一行是章节标题。
3. 建立 `资料/写作记忆/` 简版：
   - `00-当前写作状态.md`
   - `01-故事圣经.md`
   - `02-人物状态表.md`
   - `03-时间线.md`
   - `04-伏笔台账.md`
   - `05-设定规则.md`
   - `07-章节摘要.md`
   - `08-风格与禁忌.md`
   - `11-失败与避坑.md`
4. 反向解析人物、时间、地点、势力、物品、未回收伏笔和风格禁忌。
5. 写导入报告，说明解析成功项、异常章节、疑似断章、标题问题、待人工确认项。
6. 跑章节连续性和格式检查。

## 4. 反向解析口径

- 正文已经写出的内容才是事实。
- 对不确定信息标 `待确认`，不要编成设定。
- 人物状态表记录“最后已知状态”，不要按大纲替他提前变化。
- 伏笔台账区分 `open`、`advance`、`resolve`、`abandon`。
- 时间线允许先粗后细，但要能支撑续写不撞车。
- 风格与禁忌从原文抽取：段落长度、对白比例、叙述腔、常用词、禁用表达、读者预期。

## 5. 输出结构

推荐结构：

```text
正文/
资料/
资料/写作记忆/
拆文库/本书反向解析/
导入报告.md
bak/
```

如果导入用于后续 `xie-write`，必须保证 `00-当前写作状态.md` 写清下一章落点、最近风险和不能误改的事实。

可复用总入口工具：

```bash
python3 ~/.agents/skills/xie-xiaoshuo/scripts/novel_tools.py stats <项目根目录> --details
python3 ~/.agents/skills/xie-xiaoshuo/scripts/novel_tools.py check <项目根目录>
python3 ~/.agents/skills/xie-xiaoshuo/scripts/novel_tools.py doctor <项目根目录>
python3 ~/.agents/skills/xie-xiaoshuo/scripts/novel_tools.py scaffold <项目根目录> --include-entry --write
```
