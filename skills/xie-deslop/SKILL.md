# 去 AI 味

## 1. 定位

目标是让文本更像作者本人和小说现场，而不是把文字改成另一种华丽模板。默认保留原意、剧情功能、视角、信息量、人物声口和伏笔。

- 不改主线、人设、结局和章节功能。
- 不为了去 AI 味删掉有效信息、伏笔、情绪递进和人物选择。
- 用户没有授权落盘时，只输出建议或改写片段。
- 批量修正文前先备份或确认 Git 状态。

## 2. 检查门

先判断问题类型，再决定改法：

- `模板转折`：然而、不过、与此同时、另一方面、尽管如此等连续堆叠。
- `抽象总结`：用道理、主题、人生判断替代动作后果。
- `同长段落`：连续多段长度、结构、节奏过分一致。
- `列表句式`：他知道、他明白、他终于、不是因为而是因为等公式复用。
- `情绪空泛`：震惊、复杂、沉默、说不出话，但没有动作、物件、误解或选择支撑。
- `人物同声`：所有人都用同样句长、同样反问、同样清醒的总结腔。
- `章末升华`：钩子后继续解释主题、预告后文或替读者总结。
- `模型退化`：复读、截断、拒绝语、占位、括号说明、乱码、脚手架残留。

命中不等于机械删除；先复读上下文，确认它是否真的伤害阅读。

## 3. 修改原则

- 先保留信息，再换载体：把抽象解释改成动作、物件、场景反应、潜台词或误解。
- 先保留声口，再提质：将领、胥吏、商人、少年、亲眷、反派不该共用同一词源。
- 先压重复，再补现场：删除重复解释后，如损失因果，用短动作或具体细节补回来。
- 改句式不改事实：人物知道什么、不知道什么、故意隐瞒什么，不能被润色抹平。
- 章末只留钩子：最后一段以后不追加总结。

## 4. 删除比例

参考远端 `story-deslop` 的边界，但按本地项目执行：

- 局部去 AI 味：通常删改 5%-20%。
- 明显模板段：可删改 20%-35%，但必须保留信息功能。
- 大段模型退化、复读或截断：可重写受影响段落。
- 不做全章大比例重写，除非用户明确要求“重写这一章”。

如果删改会影响伏笔、人物动机或前后承接，先转为改写，不直接删除。

## 5. 批量处理

批量处理正文时，流程为：

1. 锁定范围和授权。
2. 读取项目文风、人物声口、相邻正文和待改章节。
3. 先跑可重复检查，再人工复读命中段。
4. 只改命中区域及必要承接句。
5. 回读修改后段落，确认没有断因果、掉伏笔、改声口。
6. 更新 `资料/写作记忆/11-失败与避坑.md`，记录本书高频 AI 味样本。

如果本地存在 `~/.agents/skills/story-deslop/scripts/` 中的检查脚本，可以借用其确定性检查；借用脚本不等于采用外部文风。
